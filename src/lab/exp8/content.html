Theory

grep Command

grep is an extremely powerful search utility. grep searches the specified files for the lines containing a match to the given pattern specified through a regular expression. By default, for each line that matches or contains a match to the given pattern, it prints that line as output.

The syntax for using grep command is:
grep [OPTIONS] PATTERN [FILE...]
grep [OPTIONS] [-e PATTERN | -f FILE] [FILE...]
where OPTIONS are the various options available with the grep command, PATTERN is the regular expression pattern to search for, FILE is the file containing the text which is matched against the pattern. If no file name is given then grep reads from the standard input.

The regular expression pattern should always be enclosed within single-quotes.

Regular Expressions
Regular expressions (regex) are used to specify a pattern which matches a sequence of characters given as input. The characters used for constructing regular expressions are as follows:
  • [] - anything inside the square brackets would be matched once and only once
  • Alphabet Characters - both upper and lower cases - denoted as [a-z] and [A-Z]
  • Digits denoted as [0-9]
  • ? meta character denotes a character may occur 0 or 1 times
  • * meta character denotes a character may occur 0 or more times
  • + meta character denotes a character may occur 1 or more than 1 time

For example, the strings abbde, abde, abbbbde can be denoted using a regular expression a[b]+de. The '+' meta character signifies that 'b' can be repeated more than 1 time. Similarly, a string made of alphabets and digits such as abc09af, abbbb987, a9876, abbb5 can be written using the regular expression [a-z]+[0-9]+[a-z]* where first character should be an alphabet and there may be more of alphabets, one numeric character should follow and more digits can occur there after and more alphabets may or may not occur in the string.

Thus, while using the grep command, regular expressions are very useful in searching for patterns of strings in the file. To demonstrate how grep functions, we take a sample text file and run the commands on it. The text file is as such: 
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234
ab*23
The rules of constructing regular expressions have been explained as follows:
  1. Any literal character [a-z] is a pattern that matches itself. Thus, each of the literal character is a regular expression in itself. 
    $ grep 'a' example.txt
abcdefgh
aabbaabb
abababab
ab*23
    The command matches 'a' in every line of the file. Please note that when the command is the terminal / shell, the pattern given in the command is highlighted in the output. Thus, in the above example, every occurrence of 'a' is highlighted in red color on the terminal.

    This rule doesn't apply to some meta-characters like '*','.' etc. which have special meaning. To match these meta-characters, backslash ('\') should precede them to escape the characters.

  2. Concatenation: Concatenation is achieved by writing regular expression in sequence. For example, regular expressions 'a' and 'b' can be concatenated to form a single regular expression 'ab'.

    White space characters like space are also treated as regular expressions. The strings 'ab' and 'a b' are two different regular expressions. 
    $ grep 'ab' example.txt 
abcdefgh
aabbaabb
abababab
ab*23

$ grep 'a b' example.txt
$
    All the lines with the pattern 'ab' are printed. But, for the second command, there is no output because there is not pattern 'a b' in the file.

    Parentheses '()' can be used to denote a regular expressions. 'abcd' is concatenation of four regular expressions 'a', 'b', 'c'and 'd', but '(ab)cd' is a concatenation of three regular expressions 'ab', 'c' and 'd'.

    Note: In basic regex, parentheses lose their special meaning and instead match '(' and ')'. With basic regex parentheses should be used with backslash.
    Command Output Explanation
    grep '(ab)cd' test No output Incorrect way. We are using basic regex.
    It tries to match ( and ) but none of the lines contain the parenthesis.
    grep '\(ab\)cd' test abcdefgh Correct way
    grep -E '(ab)cd' test abcdefgh Works because we are using extended regex.


    C) Bracket ([]): A list of characters enclosed within '[' and ']' matches any single character in that list.
    Command Output Explanation
    grep '[abcd]' test Example file for grep. abcdefgh aabbaabb abababab ab*23 End of example file. All of these lines contain at least one of the characters a,b,c or d


    If the first character in the list of characters enclosed within '[' and ']' is '^' then this pattern matches any character excluded from this list.

    We can use a dash (-) to specify a range of characters. [a-d] is equivalent to writing [abcd] and [1-6] is equivalent to writing [123456]. [A-Za-z0-9] matches both uppercase and lowercase letters as well as numbers.
    Command Output Explanation
    grep '[0-9]' test 123456 ABCDefgh1234 ab*23 Matches only the lines containg numbers.
    grep '[^a-z]' test Example file for grep. 123456 ABCDefgh1234 ab*23 End of example file. Matches everything except lowercase alphabets.


    D) Matching the empty string: The following operators match the empty strings.
    ^ Matches the empty string at the beginning of a line.
    $ Matches the empty string at the end of a line.
    \< Matches the empty string at the beginning of a word.
    \> Matches the empty string at the end of a word.


    Command Output Explanation
    grep '^ab' test abcdefgh abababab ab*23 Matches only the strings that start with ab.
    grep 'ab$' test abababab Matches only the strings that end with ab.
    grep '\<e' test End of example file. Matches only the words that begin with e.
    grep 'r\>' test Example file for grep. Matches only the words that end with r.


    E) Matching any character: The dot character (.) matches any single character.
    Command Output Explanation
    grep 'a...3' test ab*23 Matches any 5 character sequence that starts with a and ends with 3.


    F) Repititions: There are several repitition operators which repeat the regular expressions preceding them.
    * Matches the preceding regex zero or more times.
    ? Matches the preceding regex atmost once.
    + Matches the preceding regex at least once.
    {n} Matches the preceding regex exactly n times.
    {n,} Matches the preceding regex n or more times.
    {n,m} Matches the preceding regex at least n and at most n times.


    Note: Except '*' all of the operators described above lose their special meaning when used with basic regex. Each of these operators have to be preceded by a backslash when used with basic regex or they have to be used extended regex.

    Command Output Explanation
    grep 'a*' test Example file for grep. 123456 abcdefgh aabbaabb abababab ABCDefgh1234 ab*23 End of example file. Since all the lines contain a at least zero times, all the lines are printed as output. Appending * with a makes a optional.
    grep '^ab*' test abcdefgh aabbaabb abababab ab*23 Matches all the lines which begin with a followed by zero or more b.
    grep '^\(ab\)\(ab\)*' test abcdefgh abababab ab*23 Matches all the lines that begin with ab followed by zero or more ab.
    grep '^ab/?' test abcdefgh aabbaabb abababab ab*23 Matches all the lines which begin with a followed by zero or one b.
    grep '^ab\+' test abcdefgh abababab ab*23 Matches all the lines which begin with a followed by at least one b.
    grep '.\{10\}' test Example file for grep. ABCDefgh1234 End of example file. Matches a string exactly 10 characters long.
    grep '.\{6,\}' test Example file for grep. 123456 abcdefgh aabbaabb abababab ABCDefgh1234 End of example file. Matches a string at least 6 characters long.
    grep '.\{6,10\}' test Example file for grep. 123456 abcdefgh aabbaabb abababab ABCDefgh1234 End of example file. Matches a string at least 6 and at most 10 characters long.


    G) The OR operator (|): The OR operator (|) can be used to join two regular expressions. The resulting pattern matches any string joined by the OR operator.
    Note: To use the OR operator with basic regex precede it with a backslash.
    Command Output Explanation
    grep '\(^z+\)\|\(1234\)' test 123456 ABCDefgh1234 Matches the lines which start with z or the string 1234.

    Find Command



    The find command is used to locate files on Linux system. The command searches for the set of directory specified in the argument for the files. The syntax for find is:
    find [path] [option] [file name / expression]
    where path is the directory where the file is to be located, file name is the file to be located and expression are the regular expressions which can be used for pattern matching.

    Executing find in the terminal lists down all the files in the current directory. The command recursively searches all the files and folders in the directory. 
    $ ls
abcd.org test/

$ find
.
./test
./test/dir
./test/dir/abc.org
./test/image2.jpg
./test/image1.jpg
./test/abc.tar
./test/abc.zip
./test/abd
./test/abc.org
./abcd.org
    find command provides many options for use. In this tutorial, only two commonly used options would be explained. For further information, please refer to the man page of find.
    iname - takes an argument and recursively searches the specified directory for the given file name or pattern
    delete - sometimes, the files no longer needed have to be deleted. So, find command is used to locate them and delete is used to delete the files 
    $ find -iname abc.org
./test/dir/abc.org
./test/abc.org
    To find the files in a specified directory, 
    $ find test/ -iname abc.org
test/test/dir/abc.org
test/test/abc.org
    To search for a file using a regular expression pattern, 
    $ find test/ -iname 'ab*'
test/test/dir/abc.org
test/test/abc.tar
test/test/abc.zip
test/test/abd
test/test/abc.org
test/abcd.org
    To delete the specified file, 
    $ find test/ -iname abc.org -delete
    Please note that the delete Option should be used with caution. It should always be at the end of the command arguments. If the delete option is used before iname, all the files and folders would be deleted. The command does not ask for permission or provide any warning, whatsoever, before execution.
