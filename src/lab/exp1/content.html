File Management
Introduction

File Management includes manipulation of data and files contained in a system. One may create, delete, move, copy, rename and perform many such operations on the files and directories. This experiment provides a brief overview of some of those commands which can be used in a Linux-based environment.

Objective

The objective of this experiment is to provide a brief overview on how to create and manipulate files and directories using the Linux command-line interface.

Theory

1. The Linux File System

The file system is a method of consistently naming objects and organizing them in an orderly manner. The Linux file system is a directory tree that is consistent with each Linux distribution. This directory tree begins with the "/" (root) and then expands under that position like an upside down tree. Each directory under "/" has a specific purpose whether it is to store system files, constantly changing log files, device drivers or files created by users on the system.

A home directory is a file system directory on a multi-user operating system containing files for a given user of the system. A user's home directory is intended to contain that user's files; including text documents, music, pictures or videos, etc.

The file system also contains a method of securing files, directories and objects in the file system. This is represented by the file permissions that are explained in File Permissions

2. The Command Line Prompt

The commands that will be described are meant to be executed from a command line interface. The command line prompt that shows up in the terminal window contains useful information. It has the form:

johndoe@laptop:~/Documents$

There are three parts in the command line prompt, separated by @ : and $

The first part before the @ shows the user which is logged in at the moment. In our example it is the user johndoe. The second part between the @ and the : shows the name of the computer you are working on. In our example it is laptop. The third part, between the : and the $ shows the current directory you are working in. In our example it is ~/Documents (~ is a shortcut for the userâ€™s home directory; ~/Documents is equivalent to /home/johndoe/Documents). When you will change the current directory, you will see the third part of the command line prompt changing.

3. Viewing the contents of files and directories

We shall see how to view the contents of files and directories using three commands, namely pwd, ls and cat.

pwd command

At any time, in order to see the current directory you can type pwd(Present Working Directory).

johndoe@laptop:~/Documents$ pwd
 /home/johndoe/Documents

ls command

To list the contents of a directory, the ls command is used. When it is executed with no arguments, it lists the contents of the current working directory.

The path of a file is the location of a file in the filesystem. There are two ways of specifying the path of a file: relative path and absolute path. Relative path is the location of a file relative to your current location in the filesystem. Absolute path is the location of the file with respect to the root directory. It is independent of the current working directory.

johndoe@laptop:~/Documents$ ls my_project
vlabs_ui

Here in the above example, the relative path of the directory has been used.

johndoe@laptop:~/Documents$ ls /home/johndoe/Documents/my_project/
 vlabs_ui

Here in the above example, the absolute path of the directory has been used. As you can see, the two paths are equivalent.

The most used flags for the ls command are l, a.

FlagPurpose
lDisplays the detailed view of the contents of the folder
aAlso displays hidden contents of the folder
 
johndoe@laptop:~/Documents$ ls -l
total 4
-rw-r--r--  1 johndoe johndoe 0    2010-08-20 22:22 example.txt
-rw-r--r--  1 johndoe johndoe 0    2010-08-20 22:22 all_about_console.txt
drwxr-xr-x  2 johndoe johndoe 4096 2010-08-02 14:03 my_project

Here, in addition to the name of each file, the file type, file mode bits, number of hard links, owner name, group name, size, and timestamp, normally the modification time are printed.

The file mode bits contains details regarding the properties of the file. It has 10 fields. The first field refers to the file type. 'd' means the file is a directory, 'l' means it is a link, '-' means it is a regular file. The next nine fields are the permissions of the owner, group and other, respectively. 'r', 'w' and 'x' in a field means 'read', 'write' and 'execute' permissions, respectively. If a '-' is printed in a field, it means that particular permission is not present for the corresponding user, group or other. For detailed explanation go through File Permissions 

johndoe@laptop:~/Documents$ ls -a
.  ..  all_about_console.txt  example.txt  my_project .virtual-lab.txt

In the above example virtual-lab.txt was the hidden file, which was displayed when ls was used with -a option

. and .. refer to current folder and parent folder, respectively.

For Example if you do an ls ., it will display all the contents of the current directory

johndoe@laptop:~/Documents$ ls .
all_about_console.txt  example.txt  my_project

If you do an ls .., it will display all the contents of the parent directory

johndoe@laptop:~/Documents$ ls ..
Desktop Documents Downloads Music Pictures

ls can be used with an option "al" to list the detail view of the contents of the folder along with the hidden contents.

johndoe@laptop:~/Documents$ ls -al
total 4
-rw-r--r--  1 johndoe johndoe 0    2010-08-20 22:22 example.txt
-rw-r--r--  1 johndoe johndoe 0    2010-08-20 22:22 .virtual-labs.txt
-rw-r--r--  1 johndoe johndoe 0    2010-08-20 22:22 all_about_console.txt
drwxr-xr-x  2 johndoe johndoe 4096 2010-08-02 14:03 my_project

Whenever you need help about commands in Linux, you can read the man pages, which contain detailed information about commands and their options. To know more about the ls command, see the man page, type: 

johndoe@laptop:~/Documents$ man ls

Information about the command will appear on the screen. You can move up and down using the arrow keys. In order to exit from the man page and return to the command line type the letter q.

cat command

cat command displays the contents of the file passed as argument. If multiple files are passed, it prints the concatenated content of all the files.

       johndoe@laptop:~/Documents$ cat virtual-lab.txt
       Welcome to System and Software Lab!

In the above example cat displays content of the file virtual-lab.txt.

Type the clear command to clear your screen.

4. Moving through directories:

cd command

The command cd (change directory) allows us to move through directory trees. It has one parameter: target directory path. If parameter '-' is given it will navigate to the previous working directory. If the paramter '~' is given it will navigate to the home directory.

	  johndoe@laptop:~/Documents$ cd my_project

          johndoe@laptop:~/Documents/my_project$ pwd
          /home/johndoe/Documents/my_project

	  johndoe@laptop:~/Documents/my_project$ cd ..
	  /home/johndoe/Documents

	  johndoe@laptop:~/Documents$ cd -

	  johndoe@laptop:~Documents/my_project$ cd ~

	  johndoe@laptop:~$ pwd
	  /home/johndoe

	  johndoe@laptop:~$ cd ~james

	  johndoe@laptop:/home/james$ pwd
	  /home/james

5. Creating files and folders:

touch command

In general the text editors are able to create empty files on disk. However, sometimes it is useful to create an empty file directly from the command line. In order to do it, type:

	  johndoe@laptop:~/Documents$ touch newfile.txt

	  johndoe@laptop:~/Documents$ ls 
	  all_about_console.txt  example.txt my_project newfile.txt

An empty file with the specified name will be created. If a file with that name already exists on disk, it will remain there and moment of last update is changed in the properties of the file.

mkdir command

The mkdir command takes one or more parameters and creates the folders corresponding to the parameters. 

	  johndoe@laptop:~/Documents$ mkdir new_folder
	  johndoe@laptop:~/Documents$ ls 
	  all_about_console.txt  example.txt  my_project newfile.txt new_folder

5. Copying, Moving and removing files and folders:

cp command

cp command is used to create a copy of files. It can be invoked with two arguments, SOURCE and DEST in which case it copies SOURCE to DEST.

	  johndoe@laptop:~/Documents$ cp example.txt my_project/example_copy.txt
	  johndoe@laptop:~/Documents$ ls my_project
	  vlabs_ui  example_copy.txt

When invoked with more than two arguments, the last argument must be a directory. In this case, all the files passed as arguments are copied into the directory corresponding to the last argument.

	 johndoe@laptop:~/Documents$ cp all_about_console.txt  example.txt  newfile.txt new_folder/

	 johndoe@laptop:~/Documents$ ls new_folder
	 all_about_console.txt  example.txt  newfile.txt

For copying directories, the -r (recursive) option is used.

mv command

The mv command is used to change the location of directories and files or renaming them. Parameters are taken in a similar way as the cp command.

	 johndoe@laptop:~/Documents$ mv example.txt my_project/example_move.txt

	 johndoe@laptop:~/Documents$ ls my_project
	 vlabs_ui  example_copy.txt example_move.txt

rm command

rm command removes files. It can be used with the -r option to remove directories.

	johndoe@laptop:~/Documents$ ls 
	all_about_console.txt  example.txt  my_project  newfile.txt  new_folder 

	johndoe@laptop:~/Documents$ rm example.txt

	johndoe@laptop:~/Documents$ ls
	all_about_console.txt  my_project  newfile.txt new_folder

rmdir command

rmdir command removes empty directories.

      johndoe@laptop:~/Documents$ rmdir new_folder

      johndoe@laptop:~/Documents$ ls
      all_about_console.txt  my_project  newfile.txt

The commands cp, mv, and rm have the following commonly used parameters:

Flag Purpose
-fforces removing a file, even if user has no rights to write the file.
-iuser will be asked to confirm the operation for each file.
-bcreates backup copy of a file to be overwritten.
