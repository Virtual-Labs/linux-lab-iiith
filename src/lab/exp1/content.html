File Management
Introduction

Welcome to File Management, the first experiment of the Systems and Software lab. File Management consists of creating files and folders, copying and moving them locally etc. The goal of this experiment is to teach how to manipulate files and folders on a Linux Machine. The theory section provides a brief overview of the linux commands which are required to do file management.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports a few basic commands to enable the user to learn file management basics. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands. Then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment, go to the quiz section to test your understanding.

Objective

Learning how to create and manipulate files and directories using a Linux shell.

Pre-requisites

No pre-requisites.

Theory

1. Linux File System

A linux system is comprised of files and processes. These files are organized in a tree-like structure under the root directory.

There are different types of files in a Linux system. Most are just regular text, executable files or programs. Other types of files include directories, which are a list of files, and links, which is a mechanism to make a file accessible from a different part of a linux filesystem (discussed in detail in Advanced File Management)

The path of a file is the location of a file in the filesystem. There are two ways of specifying the path of a file: relative path and absolute path. Relative path is the location of a file relative to your current location in the filesystem. Absolute path is the location of the file from the root directory.

2. Viewing the contents of files and directories

  • To know the current working directory, enter the command 'pwd' (Present Working Directory).
  • To list the contents of a directory, the 'ls' command is used. When it is executed with no arguments, it lists the contents of the current working directory. 'ls' followed by directory path prints the contents of the specified directory.
  • 'cat' command displays the contents of the file passed as argument. If multiple files are passed, it prints the concatenated content of all the files. 

johndoe@laptop:~/Documents$ pwd
/home/johndoe/Documents

johndoe@laptop:~/Documents$ ls
example.txt  all_about_console.txt my_project

johndoe@laptop:~/Documents$ ls my_project
vlabs_ui

johndoe@laptop:~/Documents$ ls /var/
backups  cache  crash  games  lib  local  lock  log  mail  opt  run  spool  tmp

In the third example above, the relative path is used, and in the fourth one, absolute path is used.

The most used flags for 'ls' command are l, s, t, R, a.

FlagPurpose
lDisplays the detailed view of contents of the folder
aAlso display hidden contents of the folder
sDisplays the size of each of the contents of the folder
RDisplays contents recursively
tdisplays contents sorted according to modification time 
johndoe@laptop:~/Documents$ ls -l
total 4
-rw-r--r--  1 johndoe johndoe 0 2010-08-20 22:22 example.txt
-rw-r--r--  1 johndoe johndoe 0 2010-08-20 22:22 all_about_console.txt
drwxr-xr-x  2 johndoe johndoe  4096 2010-08-02 14:03 my_project

Here, in addition to the name of each file, the file type, file mode bits, number of hard links, owner name, group name, size, and timestamp ,normally the modification time are printed. The first field refers to the file type. 'd' means the file is a directory, 'l' means it is a link, '-' means it is a regular file. The next nine fields are the permissions of the owner, group and other, respectively. 'r', 'w' and 'x' in a field means 'read', 'write' and 'execute' permissions, respectively. If a '-' is printed in a field, it means that particular permission is not present for the corresponding user, group or other.

In the above example, 'my_project' is a directory (d). The owner and group is 'johndoe'. The size is 4096 bytes. Modification time is '2010-08-02 14:03'. Owner has read, write and execute (rwx) permissions, group has read and execute (r-x) permissions and others have read and execute (r-x) permissions.

By default, the size is printed in bytes. Use the "-h" option to print the size in an abbreviated, human-readable format. 

johndoe@laptop:~/Documents$ ls -a
.  ..  all_about_console.txt  example.txt  my_project

. and .. refer to current folder and previous folder. 

johndoe@johndoe-laptop:~/Documents$ ls my_project/
vlabs_ui

johndoe@johndoe-laptop:~/Documents$ ls my_project/vlabs_ui/
css  images  index.html  js  Milikan_oil_drop.html  README  simple_pendulum.html  Sys n sw.html

johndoe@laptop:~/Documents$ ls -R
.:
all_about_console.txt  example.txt  my_project

./my_project:
vlabs_ui

./my_project/vlabs_ui:
css  images  index.html  js  Milikan_oil_drop.html  README  simple_pendulum.html  Sys n sw.html

./my_project/vlabs_ui/css:
code.css  common.css  corners.css  forms.css  layout.css  reset.css

./my_project/vlabs_ui/images:
graphic-logo.gif

./my_project/vlabs_ui/js:

For more info on 'ls', type "info coreutils 'ls invocation'" on your command line.

Type the 'clear' command to clear your screen.

2. Moving through directories:

The command 'cd' (change directory) allows us to move through directory trees. It has one parameter: target directory path. If parameter '-' is given it will navigate to the previous working directory. If the paramter '~' is given iti will navigate to the home directory. 


johndoe@laptop:~/Documents$ cd my_project

johndoe@laptop:~/Documents/my_project$ pwd
/home/johndoe/Documents/my_project

johndoe@laptop:~/Documents/my_project$ cd ..

johndoe@laptop:~/Documents$ cd -

johndoe@laptop:~Documents/my_project$ cd ~

johndoe@laptop:~$ pwd
/home/johndoe

johndoe@laptop:~$ cd ~james

johndoe@laptop:/home/james$ pwd
/home/james

3. Creating files and folders:

The 'touch' command creates the files whose names are passed as parameters. If the parameter is an already existing file, it's modification time will be updated. Otherwise, the file will be created.

johndoe@laptop:~/Documents$ touch newfile.txt

johndoe@laptop:~/Documents$ ls 
all_about_console.txt  example.txt my_project newfile.txt

The 'mkdir' command takes one or more parameters and creates those folders. 


johndoe@laptop:~/Documents$ mkdir new_folder

johndoe@laptop:~/Documents$ ls 
all_about_console.txt  example.txt  my_project newfile.txt new_folder

4. Copying, Moving and removing files and folders:

  • 'cp' - It takes at least two parameters: a source file and a target location to which the file will be copied,
  • 'mv' - Used to change the location of directories and files or renaming them. Parameters are taken in similar way as the 'cp' command.
  • 'rm' - removes files/directories.
  • 'rmdir' command removes empty directories.
johndoe@laptop:~/Documents$ cp example.txt my_project/example_copy.txt

johndoe@laptop:~/Documents$ ls my_project
vlabs_ui  example_copy.txt

johndoe@laptop:~/Documents$ mv example.txt my_project/example_move.txt

johndoe@laptop:~/Documents$ ls my_project
vlabs_ui  example_copy.txt example_move.txt

johndoe@laptop:~/Documents$ ls 
all_about_console.txt  example.txt  my_project  newfile.txt  new_folder 

johndoe@laptop:~/Documents$ rm example.txt

johndoe@laptop:~/Documents$ rmdir new_folder

johndoe@laptop:~/Documents$ ls
all_about_console.txt  my_project  newfile.txt

The commands: 'cp', 'mv', and 'rm' have the following common parameters:

  • -f/force -- forces removing a file, even if user has no rights to write the file,
  • -i/interactive -- user will be asked to confirm the operation,
  • '-b' -- creates backup copy of a file to be overwritten.
