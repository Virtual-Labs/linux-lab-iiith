Status Commands
Introduction

Status commands give you all the information regarding the linux system you are working on. The goal of this experiment is to give you an understanding of how to get information about your computer system, including the users of the system, the file system space available and so on. The theory section gives a brief overview of the linux commands required to perform these tasks.

Objective

The goal of this experiment is to look at various methods at extracting information about the system and its status, including the current version, disk space, users and so on.

Theory

whoami, who and w commands

The 'whoami' command is used to find the current effective username of the user ID. 

	johnodoe@johndoe-laptop:~/$ whoami
	johndoe

The 'who' command prints information about users who are currently logged in.

	johnodoe@johndoe-laptop:~/$ who
	johndoe  tty7         2012-10-02 23:41
	johndoe  pts/1        2012-10-03 00:13 (:0.0)
	johndoe  pts/2        2012-10-02 23:44 (:0.0)
	johndoe  pts/3        2012-10-03 00:17 (:0.0)

Here, the login name, terminal line, login time and remote hostname are displayed for all the users.

The 'w' command shows the users that are currently logged in and their processes.

	 johnodoe@johndoe-laptop:~/$ w
	 00:24:37 up 45 min,  4 users,  load average: 0.05, 0.09, 0.13
	 USER     TTY      FROM              LOGIN@   IDLE   JCPU   PCPU WHAT
	 johndoe  tty7                      23:41   45:07   1:21   0.26s gnome-session --session=ubuntu
	 johndoe  pts/1    :0.0             00:13   26.00s  0.24s  0.00s pager -s
	 johndoe  pts/2    :0.0             23:44    2.00s  3.22s  2.90s vim vlab.html
	 johndoe  pts/3    :0.0             00:17    0.00s  0.22s  0.00s w

The fields displayed here are: login name, the tty name, the remote host, login time, idle time, JCPU, PCPU, and the command line of their current process.

The JCPU time is the time used by all processes attached to the tty. It does not include past background jobs, but does include currently running background jobs.

The PCPU time is the time used by the current process, named in the "what" field.

uname command

The 'uname' command is used to print certain system information. Its usage is straightforward. When invoked without any options, it prints the kernel name. Type uname --help for a complete list of options and their actions.

johnodoe@johndoe-laptop:~/$ uname --help
Usage: uname [OPTION]...
Print certain system information.  With no OPTION, same as -s.

  -a, --all                print all information, in the following order,
                               except omit -p and -i if unknown:
			         -s, --kernel-name        print the kernel name

				       -v, --kernel-version     print the kernel version
				         -m, --machine            print the machine hardware name
					   -p, --processor          print the processor type or "unknown"

					       -o, --operating-system   print the operating system
					             --help     display this help and exit
						           --version  output version information and exit

							   Report uname bugs to bug-coreutils@gnu.org
							   GNU coreutils home page:

							   For complete documentation, run: info coreutils 'uname invocation'

lsb_release command

The 'lsb_release' command prints certain LSB (Linux Standard Base) and distribution-specific information. When invoked without any arguments, it prints the LSB version. When used with the '-a' option, 'lsb_release' prints all the information. Type lsb_release -h to look at the complete set of options and their actions. 

	johnodoe@johndoe-laptop:~/$ lsb_release -h
	Usage: lsb_release [options]

	Options:
	  -h, --help         show this help message and exit
	    -v, --version      show LSB modules this system supports
	      -i, --id           show distributor ID
	        -d, --description  show description of this distribution
		  -r, --release      show release number of this distribution
		    -c, --codename     show code name of this distribution
		      -a, --all          show all of the above information
			-s, --short        show requested information in short format


	johnodoe@johndoe-laptop:~/$ lsb_release -a
	LSB Version:	core-2.0-ia32:core-2.0-noarch:core-3.0-ia32:core-3.0-noarch:core-3.1-ia32:core-3.1-noarch:core-3.2-ia32:core-3.2-noarch:core-4.0-ia32:core-4.0-noarch
	Distributor ID:	Ubuntu
	Description:	Ubuntu 12.04.1 LTS
	Release:	12.04
	Codename:	precise

df command

The 'df' command prints the amount of disk space available on the filesystem containing each file given as argument. When no argument is given, 'df' displays the amount of disk space available on all mounted file systems. 

	johnodoe@johndoe-laptop:~/$ df
	Filesystem     1K-blocks      Used Available Use% Mounted on
	/dev/sda3       63261728  41829932  18218224  70% /
	udev             1506796         4   1506792   1% /dev
	tmpfs             605632       916    604716   1% /run
	none                5120         0      5120   0% /run/lock
	none             1514080       156   1513924   1% /run/shm
	/dev/sda1         472832    102537    345881  23% /boot
	/dev/sda2      314572796 306156776   8416020  98% /dos
	/dev/sda5      104845568        64 104845504   1% /windows
	
	johnodoe@johndoe-laptop:~/$ df . /dos/Movies
	Filesystem     1K-blocks      Used Available Use% Mounted on
	/dev/sda3       63261728  41830024  18218132  70% /
	/dev/sda2      314572796 306156776   8416020  98% /dos

The '-h' option prints the disk space available in human-readable format.

	johnodoe@johndoe-laptop:~/$ df -h
	Filesystem      Size  Used Avail Use% Mounted on
	/dev/sda3        61G   40G   18G  70% /
	udev            1.5G  4.0K  1.5G   1% /dev
	tmpfs           592M  916K  591M   1% /run
	none            5.0M     0  5.0M   0% /run/lock
	none            1.5G  156K  1.5G   1% /run/shm
	/dev/sda1       462M  101M  338M  23% /boot
	/dev/sda2       300G  292G  8.1G  98% /dos
	/dev/sda5       100G   64K  100G   1% /windows

free command

The 'free' command prints the total amount of free and used physical and swap memory in the system, as well as the buffers used by the kernel. 

	johnodoe@johndoe-laptop:~/$ free
	             total       used       free     shared    buffers     cached
	Mem:       3028160    1253520    1774640          0      79576     666812
	-/+ buffers/cache:     507132    2521028
	Swap:      4194300          0    4194300

date command

The 'date' command is used to print or set the system date and time. When invoked without any arguments, 'date' prints the date and time.

cal

The 'cal' command displays a simple calendar. When invoked without any arguments, the calendar of the current month is displayed. Usage is very straightforward. Type cal --help for a complete set of options. cal -m <month> displays the calendar of the specified month. Similarly, the '-y' option displays the calenar for the specified year.
