Text Processing
Introduction

Linux provides various command line techniques for manipulating text. The goal of this experiment is to give you a basic understanding of these techniques. The theory section gives a brief overview of the various commands and tools useful in processing text from the command line, including pipes and redirection and text search and extract commands.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports a few basic commands to enable the user to learn text processing basics. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands. Then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment, go to the quiz section to test your understanding.

Theory

Pipes and Redirecction

Many shell commands take text-like input/output. It is often useful to control where this input comes from and where the output goes to, using redirection, and pass the output from one command to another's input, using pipes. Some commonly used pipes and redirects are:

>Prints the output from the command to a file on disk. If the file already exists, it is overwritten.
>>Appends the output from the command to a file on disk.
<Reads the command input from a disk file.
|Passes the output from the command on the left side of the pipe as input to the command on the right side of the pipe
Ctrl+DThe end-of-file command, signifies the end of user input. Also, when used on the command-line, exits from the shell
morePrints the ouput one screen at a time. Used for navigating through pages of output.
lessSimilar to more. Allows backward, as well as forward movement.
	johndoe@laptop:~/Documents$ ls > out.txt
	johndoe@laptop:~/Documents$ cat out.txt
	example.txt  all_about_console.txt my_project

	johndoe@laptop:~/Documents$ ls /var/ >> out.txt
	johndoe@laptop:~/Documents$ cat out.txt
	example.txt  all_about_console.txt my_project
	backups  cache  crash  games  lib  local  lock  log  mail  opt  run  spool  tmp

	johndoe@laptop:~/Documents$ pwd | ls > out1.txt
	johndoe@laptop:~/Documents$ cat out1.txt
	example.txt  all_about_console.txt my_project

grep command

The 'grep' command searches the file or standard input for lines containing matches of the given pattern, and prints these lines. The syntax is: grep [options] <pattern> [filename...] 

	johndoe@laptop:~/Documents$ ls -l | grep my_project
	drwxr-xr-x  2 johndoe johndoe  4096 2010-08-02 14:03 my_project

The following options are used to control the pattern matching:

OptionAction
-f <file>/--file=fileExtracts patterns from the file, one per line
-i/-y/--ignore-caseIgnore case in both pattern and input file
-v/--ignore-matchInvert the matching and output the lines that donot match.
-w/--word-regexpSelects lines which contain matches of the whole word only
-x/--line-regexpOnly those matches are selected that match the whole line

The following options are used to control the output:

OptionAction
-c/--countPrints the number of matching lines
-L/--files-without-matchPrints the name of each input file in which no matching instance of the pattern was found
-l/--files-with-matchPrints the name of each input file which contains at least 1 matching instance of the pattern
-m <num>/--max-count=numStop reading input file after 'num' number of matches of the pattern.
-o/--only-matchingOnly prints the matches of the pattern in the file, with each match printed on a separate line.

wc command

Prints the line, word and byte counts of each file. The syntax is: wc [option]... [file]... When invoked without any arguments, takes input from the standard input. The commonly used options are:

OptionAction
-c/--bytesprint the byte counts
-m/--charsprint the character counts
-l/--linesprint the line counts
-w/--wordsprint the word counts
 

	johndoe@laptop:~/Documents$ wc out1.txt
	1  3 45 out.txt

diff command

The 'diff' commmand compares files line-by-line. The syntax is: diff [option]... <files> Usage is fairly straightforward. Type diff --help for usage and complete list of options.

	johndoe@laptop:~/Documents$ diff out.txt out1.txt
	1a2
	> backups  cache  crash  games  lib  local  lock  log  mail  opt  run  spool  tmp

cut command

The 'cut' command prints the selected parts of each line of each input file, or standard input if no files are given. The syntax is cut [option]... [file]...

The options specify what kind of list are you specifying:

OptionAction
-bSelects specified bytes.
-cSelects specified characters.
-d <delim>Selects fields, separated by the 'delim' character.
-fSelects the specified fields.

The list of bytes, characters or fields are specified as:

  • n :-- the n'th byte, character or field.
  • n- :-- from the n'th byte, character or field till the end of line.
  • n-m :-- from the n'th to the m'th byte, character or field, including the m'th one.
  • -m :-- from the first to the m'th byte, character or field, including the m'th one.
	johndoe@laptop:~/Documents$ ls -l | cut -c 3-7 
	tal 4
	w-r--
	w-r--
	wxr-x

paste command

Each file passed as argument is printed sequentially, line-by-line separated by a TAB.

sort command

Prints the sorted concatenation of files passed as argument.

head command

Prints the first 10 lines of each file by default. The options -c and -n are used to specify the number of bytes and lines to be printed, respectively. For example, head -c 10 a.txt prints the first 10 bytes of the file a.txt.

tail command

Similar to head command. Prints the last 10 lines of each file by default. The -c and -n options behave similar to the head command.
