Advanced File Management
Introduction

Welcome to Advanced File Management, the seventh experiment of the Systems and Software lab. This experiment consists of creating hard and soft links to files and folders, manipulating them locally, learning to compress and extract files, etc. The goal of this experiment is to teach advanced file manipulation techniques on a Linux Machine. The theory section provides a brief overview of the linux commands which are required to do file management.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports a few basic commands to enable the user to learn advanced file management. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands. Then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment, go to the quiz section to test your understanding.

Theory

File Hierarchy System

The File Hierarchy Standard (FHS) defines the main directories and their contents in GNU operating systems like Linux. All files and directories appear under the root directory. Type the 'ls -a' command to view all files and folders in the current working directory, including the hidden directories. 

	johndoe@laptop:~/Documents$ ls -a
	.  ..  all_about_console.txt  example.txt  my_project

Here, '.' refers to the current directory, and '..' refers to the parent directory. 

	johndoe@laptop:~/Documents$ pwd
	/home/johndoe/Documents

	johndoe@laptop:~/Documents$ cd .
	johndoe@laptop:~/Documents$ pwd
	/home/johndoe/Documents

	johndoe@laptop:~/Documents$ cd ..
	johndoe@laptop:~/Documents$ pwd
	/home/johndoe

Shell Wildcards

Wildcards are a shell feature that allow you to select a big group of files using certain special characters. It is useful when you want to perform an operation on a particular group of files.

The following are the most commonly used wildcards in bash:

Wildcard Matches
* zero or more characters
? exactly one character
[abcde] exactly one character listed
[a-e] exactly one character in the given range.
[!abcde] any character that is not listed
[!a-e] any character that is not in the given range
{debian,linux} exactly one entire word in the options given

nano

nano is a text editor that is easy to use and offers a lot of functionality to the user. Type nano form your command line, without any arguments to open the nano editor.

There are four main sections of the editor. The top line shows the program version, the current filename being edited and whether or not the file has been modified. Next is the main window showing the main file being edited. The status line is the third line from the bottom and shows important messages. The bottom two lines show the most commonly used shortcuts in the editor.

In the shortcuts section, the Ctrl-key is represented by a '^' and can be entered either using the Ctrl-key or by pressing Esc-key twice. Type ^G to get help and ^O to save the file to disk.

Linking of Files

1. Soft or Sym linking a file:

The ln -s command is used to soft link files. A Soft link refers to symbolic path indicating the abstract location of another file. 


johndoe@laptop:~/Documents$ ln -s source_file link_name

johndoe@laptop:~/Documents$ ls -l
-rw-r--r--@  1 saint  staff  3272571 Jun 30 15:45 source_file
lrwxr-xr-x   1 saint  staff       10 Jul  1 13:56 link_name -> source_file

2. Hard Linking:

A hard linkÂ to a file is indistinguishable from the original directory entry; any changes to a file are effectively independent of the name used to reference the file. Hard links may not normally refer to directories and may not span file systems. The Command for hardlinking is ln. It can be used for files as well as directories. 


johndoe@laptop:~/Documents$ mkdir foo

johndoe@laptop:~/Documents/my_project$ mkdir bar

johndoe@laptop:~/Documents/my_project$ ln foo bar

'du' command

The 'du' command gives an estimate of the disk space usage of each file listed in the argument, recursively for directories. 'du' command entered without any arguments recursively gives the disk space usage of all files and directories in the current working directory. When arguments are entered, 'du' returns the space occupied by each of the files entered in the arguments. Some useful options are: '-c' which prints the total size occupied by the directory, '-h' which prints the file size in human-readable format. Check out the man page for a complete list of options. 

	johndoe@laptop:~/Documents/my_project/vlabs_ui$ du css
	8	css/lab
	24	css

'file' command

The 'file' is used to determine the file type of its argument. file tests each of its arguments using three sets of tests, performed in the following order: filesystem tests, magic tests and language tests. The first test that succeeds returns the file type. The type printed usually contains the one of the words, 'text', which means the file contains only printing characters and is probably safe to read on an ASCII terminal, 'executable', which means the file contains the result of compiling a program in a form understandable to some UNIX kernel or another, or 'data', meaning anything else, usually binary or non-printable. 

	johndoe@laptop:~/Documents$ file my_project
	my_project: directory

	johndoe@laptop:~/Documents$ file example.txt
	example.txt: ASCII text

tar

'tar' command is used for creating tape archives or adding and extracting files from a tape archive. The syntax is: tar -[function-letter] [options] [tarfile] [pathname ...]

Here, the function-letter specifies the function to be performed on the files. It is prefixed with a '-' or with a '--' if the long-form of the function name is used. The 'tarfile' is the name of the archive and the 'pathname' specifies the individual files.

function operation
-Aappend tar files to an archive
-ccreate a new archive
-dfind differences between archive and file system
--deletedelete from the archive
-rappend files to the end of an archive
-tlist the contents of an archive
-uonly append files newer than the copy in archive
-xextract files from an archive

Some commonly used options are:

OptionUsage
-fUse the tarfile argument as the name of the tarfile.
-vVerbose. Outputs the name of each file, preceded by the function letter.

For a complete list of options, type tar --help in your command line. 

	johndoe@laptop:~/Documents$ tar -cvf example.tar example.txt all_about_console.txt
	example.txt 
	all_about_console.txt

	johndoe@laptop:~/Documents$ tar -rvf example.tar my_project
	my_project/
	my_project/vlab_ui

zip/unzip

zip is used for compression and file packaging. unzip unpacks zip archives. The syntax is zip [options] [archive] [inpath ...] where archive is the name of a new or existing archive and inpath is a directrory or file path, optionally including wildcards. Type zip without any arguments to look at the list of options.

unzip is used to list, test and extract files from a zip archive. The syntax is unzip [options] [zipfile] [file-list] [-x xlist] [-d exdir] where zipfile is the archive path, the rest are optional arguments - file-list specifies the list of files to be extracted, xlist the list of files to be ignored and exdir the directory to which the files are to be extracted. Type unzip without any arguments to view the complete usage and options. 

UnZip 6.00 of 20 April 2009, by Debian. Original by Info-ZIP.

Usage: unzip [-Z] [-opts[modifiers]] file[.zip] [list] [-x xlist] [-d exdir]
Default action is to extract files in list, except those in xlist, to exdir;
