7. Pipes and Redirection
Introduction

Linux provides a number of commands that are focussed towards a specific and simple task. In practical applications, it is often required to perform complex tasks that involve multiple commands, with the output of one command being used to execute another command. In other words, one may need to manipulate of the input and output of these commands. For this purpose, Linux provides the powerful tool of Pipes and Redirection.

Objective

The objective of this experiment is to learn how to execute multiple commands together and manipulating the input and output of these commands. At the end of this experiment, one should be comfortable with pipes and redirection and be able to use them to manipulate the input and output of commands.

Theory

Standard Input, Standard Output, Standard Error

There are three important concepts you should know about: standard input, standard output and standard error.

Standard input (stdin) is the data taken by the program at runtime. By default, the standard input stream is the keyboard. The standard input is classified as Type 0.

Standard Output (stdout) is where data is written by the program. By default, it is the terminal from where the program is run. The standard output is classified as Type 1.

Standard Error (stderr) is the output stream where the error messages are written. It is independent of standard output. The default standard error stream is the terminal where the program was run from. The standard error is classified as Type 2.

Here, the type 0, 1 and 2 are file descriptors, which is a low-level Linux abstraction. Details of this are not required with regards to this experiment. You can simply consider type 0, type 1 and type 2 as different classifications of the input, output and error streams by the system.

Pipes and Redirection

By default, shell commands take input from stdin, write output to stdout and write error output to stderr. It is often desirable to take input from other locations, such as a file or the output of another command. Pipes and redirections are special characters or a set of special characters, which are used to redirect the input and output of commands to another location. Some commonly used pipes and redirects are: >, >>, <, |, 2> and $().

> prints the output from the command to the specified file. If the file already exists, it is overwritten. 

aditi@mylaptop$ ls
example.txt  all_about_console.txt my_project
aditi@mylaptop$ ls > out.txt
aditi@mylaptop$ cat out.txt
example.txt  all_about_console.txt my_project

>> appends the output from the command to the specified file. 

aditi@mylaptop$ cat >> out.txt
text appended at end
aditi@mylaptop$ cat out.txt
example.txt  all_about_console.txt my_project
text appended at end

Ctrl+D marks the end of file when taking input from stdin.

< reads the command input from the specified file. 

aditi@mylaptop$ cat > f1.txt
d
a
c
b
e
aditi@mylaptop$ sort < f1.txt
a
b
c
d
e

sort command is run on the contents of 'f1.txt' and the sorted output is printed to stdout.

aditi@mylaptop$ sort < f1.txt > f1_sort.txt
aditi@mylaptop$ cat f1_sort.txt
a
b
c
d
e

Here the sorted output of 'f1.txt' is printed in 'f1_sort.txt'.

2> passes the error output from the command to the specified file. Here, 2 is a reference to the file descriptor of the error stream (1 is assumed by default, so there is no need to specify the type number for stdout). 

aditi@mylaptop$ mkdir /x 2> err.txt
aditi@mylaptop$ cat err.txt
mkdir: cannot create directory `/x': Permission denied
aditi@mylaptop$ cat f1.txt 2> out.txt
d
a
c
b
e
aditi@mylaptop$ cat out.txt
aditi@mylaptop$

| passes the output from the command on the left side of the pipe as input to the command on the right side of the pipe. 

aditi@mylaptop$ cat | wc
Pipes can be
used to combine
multiple commands
to form a single
complex command.
      5      14      81

The number of lines, words and characters in the input taken from stdin is printed. 

aditi@mylaptop$ cat f1.txt
d
a
c
b
e
aditi@mylaptop$ cat f1.txt | sort
a
b
c
d
e

Prints the sorted contents of 'f1.txt'.

aditi@mylaptop$ du | more

Prints the output of the du command, one screen at a time.

$() returns the output of the command, instead of writing it to stdout. Useful for passing it as the parameter of another command. 

aditi@mylaptop$ touch $(ls -1 *.c)

Updates timestamp on all .c files in current folder

aditi@mylaptop$ wc $(ls *.txt)

Displays number of characters, words and lines in all text files in current folder.

aditi@mylaptop$ chmod 755 $(ls -1 *.org)

Changes permissions of all .org files in current folder to 755.

aditi@mylaptop$ head -n 5 $(ls -1 *.c)

Displays the first five lines (typically include statements) of all C files in current folder.

aditi@mylaptop$ tar cf "$(date +'%Y-%m-%d %H%M').tar" imp
aditi@mylaptop$ ls
example.txt  all_about_console.txt my_project imp 2012-12-31 1142.tar

Take backup of 'imp' folder in a tar file, where name of tar file reflects current date and time.

Quiz

True or False

  1. 'less ls -l' displays the output of 'ls -l' in less.
  2. 'sort < f1.txt > f2.txt' sorts the contents of f2.txt and prints the output to f1.txt.
  3. 'ls -l > output.txt' overwrites 'output.txt'.
  4. 'ls -l >> output.txt' overwrites 'output.txt'.
  5. 'sort -r -k 3 | ps aux' prints the processed sorted on CPU time.
  6. $(ls) lists the contents of the current working directory.
