Theory

Computer Networks

A computer network is a collecton of autonomous systems connected through communication links. Computer Networks facilitate the sharing of information and resources among different interconnected users.

There are various methods of classifying computer networks. The two most popular methods of classifying networks are those based on scale and topology. On the basis of scale, the computer networks can be classified into - LAN, MAN, WAN, PAN. On the basis of topology, the computer networks can be classified into - Bus Topology, Star Topology, Ring Topology, Mesh Topology and Tree Topology.

Identification of individual nodes on a network

Individual nodes in a network are identified through two parameters -

  • IP (Internet Protocol) address
  • MAC (Media Access Control) address

While the former is unique to a node, the latter is unique to the NIC (network interface card) of the node.

Representation of IP Addresses

- IPv4 :
IPv4 as the name suggests is the fourth revision of the Internet Protocol. The addresses in the IPv4 scheme are 32 bit addresses, consisting of 4 octets, written in decimal and separated by a '.' Eg. 192.168.23.42 is an IPv4 address.
- IPv6 :
Due to shortage of IPv4 addresses (only 2^32) , the Internet Engineering Task Force(IETF) designed IPv6. The new addressing scheme is of 128 bits. IPv6 addresses are written in groups of four hexadecimal digits separated by colons. Eg.2001:db8:1f70::999:de8:7648:6e8.
IPv6 addresses are logically divided into two parts: a 64-bit (sub-)network prefix, and a 64-bit interface identifier.

Representation of MAC Address

MAC addresses are represented as 6 groups of 2 hexadecimal devices separated by a colon or a hyphen. For eg. 01-23-45-67-89-ab OR 01:23:45:67:89:ab is a MAC address.

  • TX Packets - No of packets transmitted
  • RX bytes - No. of bytes received
  • TX bytes - NO. of bytes transmitted

Virtual interfaces

Assigning an IP address to the interface

The "ifconfig" command can be used to assign an IP address to the interface:

The "ifconfig" command with the interface as the first parameter, followed by the desired IP address, the keyword netmask followed by the netmask value is used to assign an IP. A sample is shown below: 

root@johndoe-laptop:~# ifconfig eth0:2 192.168.1.22 netmask 255.255.255.0

Activating and Deactivating interfaces

An interface can be brought up or down using the "ifconfig" command. "ifconfig" followed by the interface name and followed by "up"/"down" does the required job. The two commands to up(activate) and down(deactivate) the eth0:2 virtual interface are below: 

root@johndoe-laptop:~#  ifconfig eth0:2 up
root@johndoe-laptop:~#  ifconfig eth0:2 down

ip command

The 'ip' command is used for showing/manipulating routing, devices, policy routing and tunnels. The syntax is: ip [options] <object> {command | help}
The set of objects are:

ObjectMeaning
linknetwork device
addressIP or IPv6 address on a device.
routerouting table entry
rulerule in routing policy database
maddressmulticast address
mroutemulticast routing cache entry
tunneltunnel over IP

Commonly used options are:

OptionPurpose
-soutputs more information. If the option appears twice or more, the amount of information increases. The information is statistics or some time value.
-fthe option followed by the family identifier like inet, inet6 or link, enforce the protocol family to use.
-rprint the DNS names instead of host addresses using the systems name resolver.

'Command' specifies the action to perform on the object. Most objects allow you to execute the add, delete and show/list commands. The help command gives the set of all commands for an object. When the command is not specified, the default command, generally list, is executed. If the object doesnot allow list, then the help command is executed by default.

hostname command

The 'hostname' command is used to show or set the system's hostname. When called without any arguments, the current host name of the system is displayed. When followed by an argument, the hostname is changed to the given argument.

johndoe@laptop:~$ hostname
laptop

johndoe@laptop:~$ hostname vlabs
johndoe@laptop:~$ hostname
vlabs

The "ping" command

"ping" command is used mainly for network testing, measurement and management. When you type "ping" followed by the destination address on the command line, it uses the ICMP protocol's mandatory ECHO_REQUEST datagram to elicit an ICMP ECHO_RESPONSE from a host or gateway. 

johndoe@johndoe-laptop:~$ ping virtual-labs.ac.in

64 bytes from 10.4.7.5: icmp_req=1 ttl=62 time=0.493 ms
64 bytes from 10.4.7.5: icmp_req=2 ttl=62 time=0.568 ms
64 bytes from 10.4.7.5: icmp_req=3 ttl=62 time=0.454 ms
64 bytes from 10.4.7.5: icmp_req=4 ttl=62 time=0.965 ms

The output fields tell us about the host reachability, network speed and travel length.

The "traceroute" command

"traceroute" command prints the route that packets trace from an IP network to the network host. It utilizes the IP protocol's time to live (TTL) field and attempts to elicit an ICMP TIME_EXCEEDED response from each gateway along the path to the host. 

traceroute to virtual-labs.ac.in (10.4.7.5), 30 hops max, 60 byte packets
1  10.2.48.1 (10.2.48.1)  0.889 ms  1.161 ms  1.358 ms
2  10.4.4.1 (10.4.4.1)  0.638 ms  10.863 ms  11.246 ms
3  10.4.7.5 (10.4.7.5)  0.571 ms  0.928 ms  1.167 ms

ssh

ssh is a program used for logging into and executing commands on a remote machine. It provides secure encrypted communication between two untrusted hosts over an insecure network. ssh <[user@]hostname> connects and logs into the specified hostname. Usage is straightforward. Type ssh without any arguments to look at the syntax and the list of options.

Wget

Wget is a utility that facilitates the non-interactive download of files from the Web. HTTP, HTTPS and FTP protocols are supported. It can also retreive files through an HTTP proxy. The syntax is: wget [option]... [URL]... Type wget --help for its usage and complete list of options.

The -b option makes Wget work in the background. You can disconnect from the system and Wget will do the download work for you.

The -r option facilitates recursive download of files from a remote website, creating a local copy of the website, fully recreating the directory structure of the original website. This is useful for viewing websites offline.

When a download fails due to a network problem, Wget will keep retrying until it retrieves the entire file. This feature is useful when working over slow or unstable network conditions. The -c option is used to continue download of a partially downloaded file.

Another useful option is '--no-proxy' which makes Wget override any system proxy settings and download files without proxies.
