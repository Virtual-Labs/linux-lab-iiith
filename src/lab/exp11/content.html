Introduction

Welcome to Getting Help. This experiment shall help you learn the various in-built Linux features to getting help. The theory section provides a brief overview of the linux commands which are required to get help.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports a few basic commands to enable the user to learn the basics of getting help. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands. Then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment, go to the quiz section to test your understanding.

Theory

man

'man' is the system's manual pager. When you type the 'man' command followed by an argument, man searches the man pages for the term entered in the argument. Type man man to open up the man page on 'man'.

info

'info' is similar to 'man'. It was implemented as a replacement to man in the early '90s. Info pages are a more complete and comprehensive documentation than the man pages. Its usage is straightforward. Just type 'info' without any options to look at the various options and all the utilities covered. To look into the info page of a utility, just type 'info' followed by the name of whatever you are looking for.

whatis

'whatis' displays manual page descriptions. 'whatis' searches the manual page names and displays the manual page descriptions of any name matched. There is also the option of using wildcards (-w) and regular expressions (-r).

The '-h' option

Type any Linux command followed by the '-h' option to get help on the command. 

	johndoe@laptop:~$ gzip -h
	Usage: gzip [OPTION]... [FILE]...
	Compress or uncompress FILEs (by default, compress FILES in-place).

	Mandatory arguments to long options are mandatory for short options too.

	-c, --stdout      write on standard output, keep original files unchanged
	-d, --decompress  decompress
	-f, --force       force overwrite of output file and compress links
	-h, --help        give this help
	-l, --list        list compressed file contents
	-L, --license     display software license
	-n, --no-name     do not save or restore the original name and time stamp
	-N, --name        save or restore the original name and time stamp
	-q, --quiet       suppress all warnings
	-r, --recursive   operate recursively on directories
	-S, --suffix=SUF  use suffix SUF on compressed files
	-t, --test        test compressed file integrity
	-v, --verbose     verbose mode
	-V, --version     display version number
	-1, --fast        compress faster
	-9, --best        compress better
	--rsyncable       Make rsync-friendly archive

	With no FILE, or when FILE is -, read standard input.

	Report bugs to bug-gzip@gnu.org.

apropos

The 'apropos' command searches the man page names and descriptions for the search keyword. It is useful for finding the rightman pages and commands. Like the 'whatis' command, you can enter wilcards (-w) and regular expressions (-r) as keywords.

Bash autocomplete

For the lazy ones amongst you, Bash provides an autocomplete option. Once you type part of the command on the command line, just hit the [TAB] key and Bash autocompletes the rest. In case of more than one possibilities, Bash returns the set of possibilities.

  • The pwd command gives the absolute path of the current directory. True or false?
  • The latest edited file/folder comes last in the output of ls -t. True or false?
