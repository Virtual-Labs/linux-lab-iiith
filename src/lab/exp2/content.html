Experiment 2: Text Editor
Introduction

This experiment introduces a console based text editor called vim. It is an improved version of the vi editor available on most UNIX systems (vim = vi improved).

Vim is a popular tool for writing computer programs as it is very lightweight and has a lot of features such as syntax highlighting, block operations and multi-level undo. To know more about vim, visit www.vim.org.

vimtutor is a very effective tutorial for learning vim. After this experiment, learners are expected to go through vimtutor and complete the lessons there.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports few basic set of commands to enable the user learn process control basics. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands and then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment go to the quiz section to test your understanding.











Objective

This experiment aims to make the learners comfortable with editing text files (such as C programs, other source code, notes, etc.) using vim text editor.

Theory

Starting and Exiting vim

To start vim, type "vim filename" on the command prompt, e.g.

$ vim helloworld.txt

If helloworld.txt is an existing file, vim will open it. If it doesn't exist, vim will create the file.

To exit vim, go to the Escape mode (press the 'Esc' key), press the : key (Shift + ;) and type one of the following q (if you haven't made any change) q! (if you have made changes, but don't want to save it) wq (if you have made changes and want to save it) 

:wq

Modes in vim

INSERT Mode: This mode is where you will spend most of your time in vim, as it allows you to write text in the current file.

Press 'i' in Escape Mode to enter Insert Mode. When in insert mode, you will see the word "INSERT" near the bottom of the editor window, as below. 

-- INSERT --

Escape Mode or Normal Mode: This is the default mode that you get after you start vim. You can press 'i' to enter the Insert mode and start typing. Press 'Esc' key to go back to escape mode.

VISUAL Mode: This is a less-used mode. It allows selecting a section of text and applying vim commands on the selection such as cut, copy. Press 'Esc' key to go back to escape mode.

When in visual mode, you will see the word "VISUAL" near the bottom of the editor window, as below. 

-- VISUAL --

REPLACE Mode: This is also a less-used mode. It allows you to replace current text in the file with anythin you type. Press 'R' (Shift + r) to enter the REPLACE mode. Press 'Esc' key to go back to escape mode.

When in replace mode, you will see the word "REPLACE" near the bottom of the editor window, as below. 

-- REPLACE --

Cut, Copy, Paste

All these commands are run in Escape mode. Press 'Esc' key to go to Escape mode. None of the commands you type here will be visible. You will see only their effects (i.e. text being cut or copied and pasted).

command description
dw this cuts the word at the current cursor position
d2w, 2dw each of this cuts two words. You can use any number e.g. d5w cuts 5 words
d$ cuts all text from the cursor position to the end of the current line
dd cuts the current line (the line where the cursor is placed)
d5d, 5dd cuts five lines (current line + 4 lines below it)
yy copies current line so that it can be pasted somewhere else
2yy, y2y copies two lines
p pastes copied or cut content at the cursor position

When you cut more than 2 lines, e.g. 5dd, you get a message at the bottom of the vim screen, as below. 

	    5 fewer lines

Similarly, when you copy more than 2 lines, e.g. 10yy, you get this message near the bottom of the vim screen: 

	    10 lines yanked

If you run the paste command (type p in escape mode), you get this message: 

	    10 more lines

Undo, Redo

All these commands are run in Escape mode. Press 'Esc' key to go to Escape mode.

command description
u Undo
Ctrl-R Redo a change that was undone using undo command

You can undo subsequent changes using pressing 'u' again. If there are no more changes to undo, you will get the below message near the bottom of the vim window. 

	    Already at oldest change

Similarly, you can redo multiple times by repeatedly pressing Ctrl-R. When there are no more changes to redo, you will get this message: 

	    Already at newest change

Search

All these commands are run in Escape mode. Press 'Esc' key to go to Escape mode.

command description
/hello This will search for the first occurence of the word "hello" after the cursor position in the current text file.
/hello\c This will search for the word hello ignoring the case i.e. Hello, HELLO, hElLo, hello all four will match this search
n Takes you to the next occurence of the search term
N Takes you to the previous occurence of the search term

You can keep searching for the next occurrence of the search term by repeatedly pressing n after starting the search. When the search hits the bottom of the file, it will start searching from the beginning of the file. You will see the below message when that happens. 

	    search hit BOTTOM, continuing at TOP

Substitution

command description
:s/oldtext/newtext substitutes oldtext with newtext in the current line
:%s/oldtext/newtext/g substitutes oldtext with newtext in the whole document ('g' stands for global)
:%s/oldtext/newtext/gc substitutes oldtext with newtext asking for confirmation for each occurence

The global substitution gives a count of substitutions after it is done. 

60 substitutions on 60 lines

The global substitution with confirmation (gc) asks for confirmation for each occurence of the substution term. You can respond with y if you want to substitute that occurence, n if you don't want to, a to substitute all subsequent occurences, q to quit substituting. 

replace with enter (y/n/a/q/l/^E/^Y)?

This concludes the theory section of the text editor experiment. To learn more vim commands, go through the built-in vim tutorial by running the command "vimtutor" 

	    $ vimtutor

vimtutor provides a reasonably detailed overview of vim commands and helps you learn by doing. We strongly encourage you to use vimtutor for getting more familiar with vim text editor.
