Systems and Software Virtual Lab
Process Management
Introduction

Welcome to Process Control, the second experiment in the Systems and Software Virtual Lab. Process control refers to creating, killing, changing priorities of processes on a machine. The goal of this experiment is to teach basic process control on a Linux machine. The theory section provides a brief overview of commands needed to do process control.

The experimental platform for this experiment is a command line interface which mimics the terminal of a linux machine. The command line interface supports few basic set of commands to enable the user learn process control basics. There are a series of tasks which need to be performed in each experiment.

Proceed to the theory section to get an overview of the commands and then move to the experiment section to get some hands-on experience and finish the experiment tasks. After you are done with the experiment go to the quiz section to test your understanding.

Theory

What is a process?

A process, also known as task, is an executing instance of a program.

1. Viewing the processes:

the 'ps' command

'ps' command is the short form of process status. 'ps' displays information about a selection of the active processes when it is executed without any flags. 

johndoe@laptop:~/Documents$ ps
   PID   TTY        TIME    CMD
  2635   pts/9    00:00:00  ps
  26439  pts/9    00:00:00  bash

Here, PID refers to process ID, which is a unique ID for each process. TTY (terminal type) is the name of the console or terminal that the user logged into, and can also be found using the 'tty' command. TIME is the amount of CPU time for which the process has been running. CMD is the name of the command that launched the process.

The most used flags for 'ps' command are A, e, p, C, o.

FlagPurpose
ADisplays all the processes running on a system
eSame as 'A' option
pDisplays processes whose process numbers occur in pidlist ("ps -p pidlist")
CDisplays processes corresponding to a particular command
oDisplays processes in the specified format.

The first few lines display the system summary. The uptime, process information, CPU and memory state is displayed. The display of these can be toggled using single-key commands.

The first line displays the time for which the system has been up and the three load averages of the system. The load averages are the average number of processes ready to run during the last 1, 5 and 15 minutes. The uptime command gives the same output as this line. The display of uptime can be toggled by typing l.

The second line displays information about the number of processes.

The third line displays information about CPU states. Shows the percentage of CPU time in user mode, system mode, niced tasks, iowait and idle. The display of processes and states can be toggled by typing t.

The fourth line displays the memory usage statistics, including total available memory, free memory, used memory, shared memory, and memory used for buffers.

The fourth line displays statistics on swap space, including total swap space, available swap space, and used swap space. The display of Mem and Swap can be toggled typing m. The free command displays the same output as that of Mem and Swap.

The columns display the following information:

Field Meaning
PID The process ID of the process.
USER The user name of the process' owner.
PR The priority of the process.
NI The nice value of the process.
VIRT Virtual size of the process, the sum of the memory which the process is using.
RES Resident size, the amount of physical memory the process is using.
SHR The amount of VIRT size that is sharable.
%CPU The process' share of CPU time.
%MEM The process' share of physical memory.
TIME The total CPU time the process has used since it started.
COMMAND The command that started the process.

The processes are by default sorted on the basis of highest %CPU value. The processes which take up maximum CPU time are displayed on screen. To sort the processes based on some different field, single-key commands can be used. Typing M sorts the processes based on memory usage. T sorts the processes based on the total running time. N sorts the processes on the PID value. P sorts processes on CPU usage time. Sorting is done in the decreasing order in all cases. Also, the commands are case-sensitive.

The number of processes displayed by default depends on the screen size. We can change this by typing the n command, which will prompt you to enter the number of processes to be displayed. Setting this to '0' will will reset the default behaviour.

The time interval in which the display is updated is called the 'delay time'. Type s or d command to change the delay time. Hit space to update the ouput immediately.

Type q to quit.

2. Control Codes

The shell offers a number of control codes that perform a specific command. A control code is issued by holding down the Ctrl key, followed by a key-press.

Control Code Action
Ctrl+cCancels a command or interrupts a running program.
Ctrl+zSuspends a process or job, without terminating. "fg" command can be used to restart the suspended job or process

3. Running processes in the background

When we use the shell to launch a process, we wait for the process to be completed before we get back the control of the shell.This is called running a process in the foreground. It is also possible to run a process in the background. In this case, the program is launched and control is returned the the shell immediately, allowing you to run other commands. In the meantime, the initial process continues to run in the background. To place a process in the background, we append '&' at the end of the command. 

johndoe@laptop:~/Documents/my_projects$ gedit filename.txt &
johndoe@laptop:~/Documents/my_projects$

We can use the 'jobs' command to view the background processes.

johndoe@laptop:~/Documents/my_project$ jobs
[2]+  Done                    gedit filename.txt

If a process running in the foreground is taking a long time, we can relegate it to the background. This is a 2 step process.
a). Hold Ctrl + Z key to suspend the process.
Use "jobs" to see the current status of the process.

johndoe@laptop:~/Documents/my_project$ jobs
[2]  + Suspended        gedit filename.txt

b) Now type in "bg %N" where N is the job id - 2 in this case. 

johndoe@laptop:~/Documents/my_project$ bg %2
[2] gedit filename.txt &

Now control will return to the shell and the process will execute in the background.
To bring back the process to the foreground, we use the 'fg' command.

johndoe@laptop:~/Documents/my_project$ fg %2

Now the shell prompt goes away and control is given back only after process completes the execution.

4. Killing processes

We use the "kill" command to kill the processes.The "kill" command takes the PID of the process as an argument.
Suppose we want to kill the process 'gedit' 

johndoe@laptop:~/Documents$ ps -e | grep gedit
3155 ?        00:00:05 gedit

The first column of the output gives the PID of the gedit process 

johndoe@laptop:~/Documents$ kill 3155 
johndoe@laptop:~/Documents$

We can now check whether the process has been killed or not.

johndoe@laptop:~/Documents$ ps -e | grep gedit
johndoe@laptop:~/Documents$

Since there is no output, it means that there is no process running with the name gedit, which means that it has been killed.

5. Changing Process Priority:

The priority of a process is indicated by its "nice value". Lesser the nice value of a process, more is the priority. Nice value ranges from -20 (fastest) to 20 (slowest). The nice value of a process (pid = 100) can be viewed by using the ps command as: ps -o pid,nice -p 100.
The nice value of a process can be changed by using the renice command. The renice command takes two arguments: the new priority and process ID. 

johndoe@laptop:~/Documents$ ps -o pid,nice -p 2292
PID NI
2292 0
johndoe@laptop:~/Documents$ renice -10 2292
2292: old priority 0, new priority -10
johndoe@laptop:~/Documents$
